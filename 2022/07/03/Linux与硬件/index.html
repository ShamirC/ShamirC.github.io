与 widows 不同， Linux 中硬件是一个个 文档

硬件装置在 Linux 的档案名

Linux 中几乎所有的硬件装置文件都在 /dev 目录下

常见的硬件在 Linux 中的文件名

裝置 裝置在Linux內的檔名
SCSI/SATA/USB硬碟機 /dev/sd[a-p]
USB快閃碟 /dev/sd[a-p] (與SATA相同)
VirtI/O界面 /dev/vd[a-p] (用於虛擬機器內)
軟碟機 /dev/fd[0-1]
印表機 /dev/lp[0-2] (25針印表機) /dev/usb/lp[0-15] (USB 介面)
滑鼠 /dev/input/mouse[0-15] (通用) /dev/psaux (PS/2界面) /dev/mouse (當前滑鼠)
CDROM/DVDROM /dev/scd[0-1] (通用) /dev/sr[0-1] (通用，CentOS 較常見) /dev/cdrom (當前 CDROM)
磁帶機 /dev/ht0 (IDE 界面) /dev/st0 (SATA/SCSI界面) /dev/tape (當前磁帶)
IDE硬碟機 /dev/hd[a-d] (舊式系統才有)

磁盘的（物理）分区

磁盘由 磁碟盘，机械手臂，磁碟读取头 与 主马达

磁碟盘上又可使用两种单位进行细分：

  • 磁区
  • 磁轨

早期磁碟分割中，第一个磁区包含了重要的信息（称为MBR,Master Boot Record格式)。此外，随着磁碟的容量越来越大，出现了一种新的磁区分割格式，GPT (GUID partition table)

MSDOS(MBR) 与 GPT 磁碟分割表

如上图所示，所有磁盘的同一磁轨称为磁柱（Cylinder)，通常是存储的最小单元。也有使用磁区作为最小分割单位的。

MSDOS

早期 Linux 为了兼容 Windows, 采用了 MBR（Master Boot Record) 的方式处理开机管理软件与分割表，这些放在磁碟的第一个磁区，这个磁区通常是 512 bytes ：

  • MBR，占 446 bytes，安装开机管理软件的地方
  • 分割（区）表，记录整个硬盘碟分割的状态，占 64 bytes

上图所示为一个分区的展开（假设有400个磁柱），分割（区）表所在区域仅有 64 bytes 容量， 因此最多仅能有 4 组记录区，每组记录区记录了该区段的起始与结束的磁柱号码。

假设上面的硬盘装置文件名为 /dev/sda 时，那么上图中（p1-p4) 4 个 分割槽在 Linux 系统中的文件目录如下：

  • P1:/dev/sda1
  • P2:/dev/sda2
  • P3:/dev/sda3
  • P4:/dev/sda4

在 windows 中, p1-p4 这 4 个分割槽的代号应该是 C,D,E,F。当你有资料要写入F 盘时，实际上是指你的资料被写入这个硬盘的 301-400 号磁柱之间。其实所谓的分区，只是对 64 bytes 的分割 表进行设定

虽然分割表只有记录 4 组资料的空间，但是一个硬盘甚至可以被分割成 10 个以上的分区。方法是利用额外的磁区来记录更多的分区信息

延伸分割（Extented )

上图记录中的 P2 为 延伸分割，延伸分割的主要目的是使用额外的磁区记录分割信息，延伸分割本省并不能被格式化，我们可以通过延伸分割所指向的磁区继续进行分割的记录。如上图所示，L1-L5 是由延伸分割继续切出来的分割槽，被称为 逻辑分割

  • 主要分割 (primary) 与延伸 (extented) 分割最多可以由 4 个
  • Extented 只能有一个（操作系统限制）
  • 逻辑分割是由延伸分割持续切出来的分割槽

GUID partition table, GPT 磁碟分割表

GPT 以 逻辑区块位址（Logical Block Address, LBA)，来规划磁碟（预设位 512 bytes), 与 MBR 仅使用第一个 512 bytes 区块来记录不同，GPT 使用了 34 个 LBA 来记录，整个磁碟最后 33 个 LBA，下图LBA-34~LBA-1 也拿来做备份。

LBA 0: 储存第一阶段的开机程序，以及一个表示 GPT 分割表的标志

LBA 1: 表头记录区，记录了分割表本身的位置与大小，同时记录了备份用的 GPT 分割（LBA-1~LBA-34) 的位置，分割表的检验机制码（CRC32), 操作系统可以根据检验码来判断 GTP 是否正确

LBA 2 ~ LBA 33 (实际记录分割信息）:从 LBA2 开始，每个 LBA 都可以记录 4 笔分割记录，所以在预设的 情况下，总共有 324 = 128 笔分割记录，每个 LBA 有 512 bytes，因此每笔记录用到了 128 bytes，每笔记录中提供了 64 bits 来记载 开始/结束 的磁区号码。 **因此，GPT 分割表对于单一分割槽来说，最大容量限制为 [ 2^64 512 bytes = 2^63 1 Kbytes = 2^33 TB = 8 ZB ]** ，

开机流程中的 BIOS 与 UEFI 开机检测程序

操作系统控制所有硬件，并且提供核心功能，这样电脑就能够识别硬盘内的文档系统，并进一步读取硬盘内的软件文档并执行该软件来完成软件的执行目的。

既然 操作系统 也是一个软件，那么电脑是如何识别并执行它的呢？

目前的主机系统在载入硬件驱动的程序，主要有早期的 BIOS 与 UEFI 两种机制

BIOS 搭配 MBR/GPT 的开机流程

CMOS:

​ 记录各项硬件参数且嵌入在主板上的储存器

BIOS：

​ 嵌入主板上的固体。电脑系统会在开机时主动执行这一程序

  1. 执行 BIOS，BIOS 依据使用者的设定获得能够开机的硬盘
  2. 读取可开机硬盘的第一个磁区的 MBR 的位置，MBR 这个仅有 446 bytes 的硬盘容量里面会放置最基本的开机管理程序
  3. 开机管理程序 （Boot loader）是安装操作系统时提供的，一个可读取核心档案来执行的软件
  4. 核心档案：开始操作系统的功能

目录树架构中最重要的就是 根目录，用 /表示

档案系统与目录树的关系（挂载）

挂载是指利用一个目录当成进入点，将磁盘槽的资料放置在该目录下，进入该目录就可以读取该分区槽，这个动作就叫 挂载，进入点的目录称为 挂载点
