Linux 档案权限与目录配置

Created2022-07-29|Updated2023-08-17|Linux
|Post View:

使用者与群组

档案拥有者(User)

Linux 允许多人工作，因此，一些个人文档可以设置档案拥有者，这样可以保护隐私权以及每个人喜好的工作环境

群组(Group)

Linux 可以设定 群组，一个群组有多个成员，通过档案权限设定，可以限制非群组的成员浏览修改档案。

其他人（Others）则是对于群组成员来说的团队外的成员

Linux 使用者身份与群组记录的档案

在预设情况下

  • 所有的系统上的账号与一般身份使用者，还有 root 的相关资讯，都记录在 /etc/passwd
  • 个人密码记录在 /etc/shadow
  • Linux 所有的群组名称 都记录在 /etc/group

Linux 档案权限

档案属性

ls -al 指令查看档案

上面 7 栏分别为：档案类型权限，连结数，档案拥有者，档案所属群组，档案容量(默认为bytes)，档案最后被修改时间，档案名

第 一 栏 第一个字符为档案类型权限：

  • 第一个字符，[d] 则表示这个档案是 目录
  • 当第一个字符为 [-], 则是 档案
  • 若第一个字符为 [l] ， 则表示为 连结档（link file)
  • 若第一个字符为 [b] , 则表示为装置档案里面的可共存储的周边设备
  • 若第一个字符为 [c] , 则表示为装置档案里的键盘，鼠标等一次性读取装置

第 一 栏 接下来的字符中，已三个为一组，且均为 [rwx]的三个参数组合, 三个权限的位置不会改变，如果没有对应权限，就会出现 [-]

  • r 代表可读 （read), w 代表可写 （write), x 代表可执行 (execute)
  • 三组 [rwx] 分别为 档案拥有者 可具备的权限，加入此群组账号的权限，非本人且没有加入本群组的其它账号的权限

改变档案属性与权限

  • chgrp : 改变档案所属 group
  • chown: 改变档案 user
  • chmod: 改变档案的权限， SUID, SGID, SBIT 等等的特性
